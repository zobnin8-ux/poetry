## 1. Краткое резюме

Создан **сайт-книга** — электронное собрание стихов с концепцией «книга на экране»: тёплая типографика, оглавление по главам, разворот с иллюстрацией слева и стихом справа, каркас для добавления десятков и сотен стихов без переделки дизайна.

Проект **не дублирует** Стихи.ру, а служит **главной витриной** собрания: полные тексты, иллюстрации художника, навигация как в томе. Ссылка на Стихи.ру указана на странице автора.

**Сейчас опубликовано на сайте:** 5 стихов, 2 с готовыми иллюстрациями, 3 с заглушкой «художник рисует». Заложено **8 глав** оглавления (большинство пока пустые — готовы к наполнению).

## 2. Идея и цели проекта

### 2.1. Замысел

| Элемент | Описание |
|--------|----------|
| **Название** | «Жизнь напишу карандашом» — заглавный стих и имя всего собрания |
| **Формат** | Книга на экране, а не лента постов и не копия портала |
| **Тон** | Тёплый, спокойный, литературный; без медийного шума |
| **Иллюстрации** | Параллельная линия художника; рисунки добавляются по мере готовности |
| **Язык** | Только русский |
| **Аудитория** | Открытая — «всем, кому интересно» |

### 2.2. Что сознательно не входило в концепцию сайта

- Раздел про «терапевтический» или «исцеляющий» эффект стихов (обсуждалось в переписке, на сайт **не выносится**).
- Коммерция, подписки, агрессивные призывы к действию.
- Автоматический парсинг текстов со Стихи.ру (права и контроль качества — у автора; тексты добавляются вручную в файлы).

### 2.3. Цели, которые проект уже выполняет

1. Дать **рамку книги** под ~50–80+ стихов.
2. Показать **пары «стих + иллюстрация»** там, где они есть.
3. Показать **заглушку** там, где рисунок в работе.
4. Позволить автору **добавлять стихи** без программирования (файлы Markdown).
5. Дать **оглавление по главам**, вход для новых читателей, страницу художника.

## 7. Система контента (как добавлять стихи)

### 7.1. Один стих = один файл Markdown

Путь: `content/poems/imya-fayla.md`  
URL: `/stih/imya-fayla` (имя файла без `.md`).

### 7.2. Шапка файла (YAML)

```yaml
---
title: "Название стиха"
chapter: nachalo          # id из chapters.ts
order: 1                  # порядок внутри главы
illustration: none        # none | pending | ready
image: /illustrations/file.png   # только при ready
featured: false           # опционально
---
```

### 7.3. Тело стиха — критически важно

- **Каждая строка стиха — с новой строки** в файле.
- **Пустая строка между строфами** — иначе строфы слипнутся в один блок (эта ошибка уже была исправлена в коде).

Пример:

```
Первая строка
Вторая строка
Третья строка
Четвёртая строка

Вторая строфа
...
```

### 7.4. Состояния иллюстрации

| Значение | Поведение на сайте |
|----------|-------------------|
| `none` | Только текст; узкая страница-лист |
| `pending` | Текст + блок-заглушка: «Художник работает над иллюстрацией…» |
| `ready` | Разворот книги; файл в `public/illustrations/` |

### 7.5. Добавление картинки

1. Положить PNG/JPG/SVG в `public/illustrations/`.
2. В шапке стиха: `illustration: ready` и `image: /illustrations/имя.png`.
3. Закоммитить и отправить на GitHub — Vercel пересоберёт сайт.

## 8. Дизайн и вёрстка

### 8.1. Цветовая палитра

| Переменная | Назначение | Пример |
|------------|----------|--------|
| `--color-parchment` | Фон сайта | #f6f1e8 |
| `--color-parchment-deep` | Линии, глубина | #ebe4d6 |
| `--color-ink` | Текст стиха | #2c2c2c |
| `--color-ink-muted` | Автор, подписи | #5c5348 |
| `--color-accent` | Ссылки, колонтитулы | #8b7e6a |

Иллюстрации **не перекрашиваются** — цвет даёт только картина; UI остаётся сдержанным.

### 8.2. Типографика

- Шрифт: **Literata** (Google Fonts, кириллица).
- Стих: строки блочные (`.poetry-line`), строфы с отступом (`.poetry-stanza`).
- В развороте книги: чуть компактнее кегль, чтобы длинные стихи помещались на лист.

### 8.3. Разворот книги (десктоп, ≥1024px)

Реализован компонентом `BookSpread` и стилями в `globals.css`.

**Структура:**

```
┌─────────────────────────────────────────────────────────┐
│  «Стол» — фон book-desk (темнее, тень)                  │
│   ┌──────────────────┬──┬──────────────────┐            │
│   │ Левая страница   │▓▓│ Правая страница  │            │
│   │ (иллюстрация)    │g │ заголовок        │            │
│   │                  │u │ автор            │            │
│   │                  │t │ стих по центру   │            │
│   │                  │t │ листа, строки    │            │
│   │                  │e │ слева в блоке    │            │
│   └──────────────────┴──┴──────────────────┘            │
└─────────────────────────────────────────────────────────┘
```

**Принятые решения (по вашим пожеланиям):**

1. **Обе страницы одного формата** — одинаковый фон «бумаги», одна минимальная высота (~88vh).
2. **Переплёт (gutter)** — ~26px, градиент и внутренние тени, чуть выразительный.
3. **Прокрутка** — весь разворот на странице; правая часть растёт со стихом; **внутри** правого листа отдельного скролла нет.
4. **Ширина** — до ~1360–1420px на широких экранах.
5. **Стих на правом листе** — блок **по центру страницы**; строки **выровнены по левому краю** внутри узкой колонки; поля сверху/снизу ~3.25–3.5rem.

**Мобильный вид:** два отдельных «листа» друг под другом (иллюстрация, затем текст), не сжатые колонки.

### 8.4. Страница стиха без иллюстрации

Класс `.poem-sheet`: заголовок и автор по центру, тело стиха — центрированный блок с левым выравниванием строк (как в книжной колонке).

## 10. История разработки (этапы)

Обсуждение и реализация шли в диалоге с автором. Основные вехи:

1. **Концепция** — «книга на экране», название «Жизнь напишу карандашом», параллельная линия художника, без акцента на «терапию».
2. **Каркас** — Next.js, главы, файлы стихов, заглушки иллюстраций.
3. **Первые стихи** — 4–5 текстов, ссылка на Стихи.ру.
4. **GitHub** — репозиторий `zobnin8-ux/poetry`, первый push.
5. **Vercel** — публичный URL poetry-taupe.vercel.app.
6. **Иллюстрации** — «Черная душа», «К новой жизни».
7. **Вёрстка разворота** — итерации: колонки → книга с переплётом → шире → строфы → центрирование стиха на листе.

### Коммиты (хронология по смыслу)

- Добавить сайт-книгу «Жизнь напишу карандашом».
- Иллюстрация к «Черной душе».
- Разворот: иллюстрация слева, стих справа.
- Разворот книги: равные страницы, переплёт.
- Шире разворот, стих целиком на правом листе.
- Строфы и строки (исправление «единого блина» текста).
- Стих по центру страницы с полями.

## 15. Итог

Проект **«Жизнь напишу карандашом»** — это рабочий прототип **полноценной электронной книги** с профессиональной задумкой (разворот, главы, иллюстрации, задел на 80 стихов). Технически он прост в сопровождении: контент в Markdown, сайт на Next.js, хостинг на Vercel.

Главная ценность на текущем этапе — **не количество стихов на сайте (5)**, а **готовая архитектура и оформление**, в которую можно спокойно вкладывать всё собрание и иллюстрации по мере готовности — без смены концепции и без «ломания» дизайна.
