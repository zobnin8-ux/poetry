# Жизнь напишу карандашом

**Сайт-книга** — электронное собрание стихов **Андрея Зобнина** с иллюстрациями соавтора-художника. Тёплая типографика, оглавление по главам, разворот «иллюстрация слева — стих справа», каркас под десятки и сотни произведений без смены концепции.

## Идея проекта

| Элемент | Описание |
|--------|----------|
| **Название** | «Жизнь напишу карандашом» — заглавный стих и имя всего собрания |
| **Формат** | Книга на экране, а не лента и не копия Стихи.ру |
| **Тон** | Спокойный, литературный, без медийного шума |
| **Иллюстрации** | Добавляются по мере готовности художником |
| **Язык** | Только русский |

**Сознательно не входит в концепцию сайта:** раздел про «терапевтический» эффект стихов; коммерция и агрессивные CTA; автоматический парсинг текстов со Стихи.ру (тексты добавляются вручную в Markdown).

**Макеты (mockup):** используются только как **ориентир вёрстки** (пропорции, обложка, разворот). Тексты и чужие иллюстрации с макетов **не** подставляются — только стихи автора и файлы из `public/illustrations/`.

### Интерфейс

- **Главная (`/`)** — простая титульная заглушка (название, автор, «Открыть книгу»). **Обложка по макету** — в обсуждении, не свёрстана.
- **Разворот** (десктоп, стихи с `pending` / `ready`) — открытая книга: корешок, боковые «срезы» блока страниц, тень под книгой, орнаменты, без строки автора на правом листе, вписывание в экран без внутреннего скролла листа.
- **Мобильный** — два «листа» друг под другом (иллюстрация, затем текст).

## Быстрый старт

```bash
git clone https://github.com/zobnin8-ux/poetry.git
cd poetry
npm install
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

| Команда | Действие |
|---------|----------|
| `npm run dev` | Режим разработки |
| `npm run build` | Продакшен-сборка |
| `npm run start` | Запуск после `build` |
| `npm run lint` | ESLint |

> **Примечание:** на некоторых машинах Windows `npm install` может падать с ошибкой TLS (`UNABLE_TO_VERIFY_LEAF_SIGNATURE`). Сборка на **Vercel** при этом проходит успешно.

## Как добавлять контент

### 1. Новый стих

1. Скопировать `content/poems/_SHABLON.md` → `novyj-stih.md`.
2. URL будет `/stih/novyj-stih` (имя файла без `.md`).
3. Заполнить шапку и текст под `---`.

**Шапка (YAML):**

```yaml
---
title: "Название стиха"
chapter: nachalo          # id из content/chapters.ts
order: 10                 # порядок внутри главы
illustration: none        # none | pending | ready
# image: /illustrations/novyj-stih.png   # при ready
featured: false
---
```

**Тело — важно:**

- каждая **строка стиха** — с новой строки в файле;
- **пустая строка** между строфами (иначе строфы сольются).

### 2. Иллюстрация

| `illustration` | Поведение |
|----------------|-----------|
| `none` | Узкая страница только с текстом |
| `pending` | Разворот + тёплая заглушка «художник рисует» |
| `ready` | Разворот + картинка из `public/illustrations/` |

Шаги для готовой картинки:

1. Положить файл в `public/illustrations/`.
2. В шапке: `illustration: ready` и `image: /illustrations/имя.png`.
3. `git push` → Vercel пересоберёт сайт.

### 3. Настройки сайта

`content/site.json`:

```json
{
  "title": "Жизнь напишу карандашом",
  "author": "Андрей Зобнин",
  "subtitle": "собрание стихов",
  "artistName": "Художник",
  "artistNote": "...",
  "stihiUrl": "https://stihi.ru/avtor/boeing75",
  "startSlugs": ["zhizn-napishu-karandashom", "..."]
}
```

### 4. Главы

`content/chapters.ts` — римские номера и id:

| Рим. | id | Название |
|------|-----|----------|
| I | `nachalo` | Начало |
| II | `svet` | Свет и любовь |
| III | `ten` | Свет и тень |
| IV | `blizkie` | Близкие |
| V | `doroga` | Дорога |
| VI | `gorod` | Город и день |
| VII | `tajna` | За порогом |
| VIII | `pamyat` | Память |

Пустые главы в оглавлении показывают подсказку «добавьте стихи».

### 5. Нумерация страниц

Сквозной номер листа считается автоматически: `getPoemPageNumber()` в `src/lib/content.ts` — по общему порядку стихов в книге (глава → `order`).

## Дизайн и разворот книги

### Палитра (`src/app/globals.css`)

| Переменная | Назначение |
|------------|------------|
| `--color-parchment` | Фон сайта |
| `--color-parchment-deep` | Разделители |
| `--color-ink` | Основной текст |
| `--color-ink-muted` | Подписи |
| `--color-accent` | Ссылки, колонтитулы |
| `--color-gold` | Орнаменты, номер страницы |

Иллюстрации **не перекрашиваются** — цвет даёт только файл картинки.

### Разворот (десктоп ≥1024px)

Компонент `BookSpread` + стили `.book-outer`, `.book-spread`, `.book-page`, `.book-gutter`.

```
┌ book-edge ─┬──────────────────────────────────────────────┬─ book-edge ─┐
│  (срез)    │  левый лист          │ корешок │ правый лист │   (срез)    │
│            │  иллюстрация         │ закладка│ колонтитул  │             │
│            │  нижний орнамент     │         │ заголовок   │             │
│            │                      │         │ стих        │             │
│            │                      │         │ орнамент + №│             │
└────────────┴──────────────────────┴─────────┴─────────────┴─────────────┘
         drop-shadow под книгой (без тёмной «траурной» рамки)
```

**Принятые решения:**

- обе страницы — «бумага» одного тона;
- переплёт с тёмной лентой и закладкой;
- стих на правом листе: блок по центру страницы, строки выровнены влево внутри колонки;
- **без** строки автора на развороте (как на целевом макете);
- **без** внутреннего скролла на листе — подгонка высоты и `clamp()` для кегля;
- вертикальное центрирование разворота в окне;
- на длинных стихах (например «Черная душа») — более плотная вёрстка на низких экранах.

**Мобильный:** `book-page-mobile` — иллюстрация и текст отдельными блоками.

### Стих без иллюстрации

Класс `.poem-sheet` — заголовок, автор, текст, номер страницы внизу.

| Компонент | Файл | Роль |

## Права и этика

- Тексты — **авторские**, публикуются с согласия автора.
- Иллюстрации — нужно **согласие художника** на веб-публикацию.
- Стихи.ру — **внешняя ссылка**, не встраивание; полные тексты хранятся в репозитории.
- Не использовать автоматический скрейпинг stihi.ru.
