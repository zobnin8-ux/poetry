## Идея проекта

| Элемент | Описание |
|--------|----------|
| **Название** | «Жизнь напишу карандашом» — заглавный стих и имя всего собрания |
| **Формат** | Книга на экране, а не лента и не копия Стихи.ру |
| **Тон** | Спокойный, литературный, без медийного шума |
| **Иллюстрации** | Добавляются по мере готовности художником |
| **Язык** | Только русский |

**Сознательно не входит в концепцию сайта:** раздел про «терапевтический» эффект стихов; коммерция и агрессивные CTA; автоматический парсинг текстов со Стихи.ру (тексты добавляются вручную в Markdown).

**Макеты (mockup):** используются только как **ориентир вёрстки** (пропорции, обложка, разворот). Тексты и чужие иллюстрации с макетов **не** подставляются — только стихи автора и файлы из `public/illustrations/`.

Ключевое слово проекта: **СОБРАНИЕ** (не блог, не портал, не лента).

## Полное видение автора (май 2026)

> Зафиксировано в README, чтобы при потере контекста чата (Cursor, Windows, новая сессия) восстановить замысел из репозитория. Обсуждение — без спешки; сила в **постепенном сборе книги**, не в «релизе продукта».

### Главная мысль

Сайт **не** должен выглядеть как: блог, портал стихов, соцсеть, авторский лендинг, копия Стихи.ру, AI-минимализм, инфобизнес, каталог текстов.

Сайт **должен** быть: **цифровая книга** — не PDF, а атмосфера бумаги, тишины, личного чтения, постепенного погружения. Человек чувствует не «сайт», а **старый том, который долго собирали**.

**Финальное ощущение:** тишина, глубина, присутствие автора, время, человеческая рука, «неинтернетность». Не «прикольный сайт», а «будто нашёл чью‑то настоящую книгу».

### Что уже удачно (сохранять)

| Элемент | Почему важно |
|--------|----------------|
| **Разворот** | Иллюстрация слева, стих справа, переплёт, бумага — «читаю книгу, а не листаю сайт». Главный визуальный актив. |
| **Иллюстрации** | Сборник + художественный мир (арт-книга, альбом). Стиль: карандаш, тень, фактура, ручная работа — **не** яркий AI-глянец. |
| **Главы** | 80 стихов нельзя свалить списком; нужны ритм и путь. Сейчас главы **технические** — цель: **части жизни / внутренние состояния**, не «категории». |

### Визуальная атмосфера

- Тихая литературная среда: старая библиотека, вечерний свет, письменный стол, графит, личный архив.
- Цвета: тёплая бумага, графит, серо-коричневый, приглушённый чёрный, выцветший беж.
- **Нельзя:** яркие кнопки, кислотные акценты, стекло, неон, современные градиенты, luxury-tech UI.

### Главная страница

Сейчас — вход-заглушка. **Должна стать обложкой книги:** воздух, название, возможно одна цитата, подпись автора, спокойная кнопка «Открыть книгу». **Без** длинных биографий, достижений, SEO-портянок на первом экране. Первый экран = **вход в мир** (поздний вечер, сайт не кричит).

### Анимации (если будут)

Только медленные, почти незаметные: лёгкий fade, тень, мягкое «перелистывание». **Без** parallax, летающих блоков, Apple-эффектов, 3D. Сайт **медленный по ощущению** — это сила.

### Путь читателя (целевой)

Сейчас: Главная → Оглавление → Стих.

Цель: **путешествие:**

1. **«С чего начать»** — не список файлов, а 5–7 ключевых стихов, маршрут по настроению для новичка.
2. **Главы** — этапы жизни (пример направления: Начало → Свет → Разлом → Тень → Дорога → Ожидание → Потери → Память → За порогом).
3. **Якорные стихи** — не все равны; сильные стихи: иллюстрации, «с чего начать», центр главы, главная. Не делать всё «главным».

### Ощущение автора

Сила — «написал живой человек»: не медийный, не инфоцыган, не коуч. **Просто писал.** Нельзя убить маркетингом и официозом.

**Страница `/avtor`:** тихое присутствие (фото, короткий текст, зачем стихи), **не** резюме, не список наград, не «я великий автор».

### Художник

**Второй голос книги**, не «дополнение». Иллюстрации **не** буквально объясняют стих — продолжают настроение, тайну, второй слой.

### Запреты (жёстко)

- Блог: новости, лента, активность ради активности.
- SEO-помойка: километры текста под поиск.
- Instagram-стиль: карточки, карусели, кричащие CTA.
- Пытаться понравиться всем — проект **камерный**.

### На будущее (осторожно, после «дыхания» книги)

| Идея | Условие |
|------|---------|
| Аудио | Спокойное чтение, без театра и «музыки с YouTube»; максимум тихий фон, живой голос |
| Рукописи | Фото листов, правки карандашом — **дозированно** |
| Печатная книга | Архитектура сайта уже близка к томунику; арт-тираж / подарочное издание |

## Конституция проекта — что сохранять

Перед любой правкой сверяться с этим списком:

1. Не блог, не портал, не копия Стихи.ру — **собрание**; Стихи.ру только ссылка.
2. **Разворот и бумага** — эталон; новые экраны не «современнее» разворота.
3. **Главная = обложка** — вход в мир, не лендинг.
4. **Главы = движение жизни** (со временем — названия и короткие вступления).
5. **Не все стихи равны** — якорные, `startSlugs`, `featured`, иллюстрации.
6. **Художник = второй голос**, не иллюстратор «к тексту».
7. **Медленно, камерно**, без SEO-помойки и кричащих CTA.
8. **Собирать книгу**, не «релизить стартап».

## Как добавлять контент

### 1. Новый стих

1. Скопировать `content/poems/_SHABLON.md` → `novyj-stih.md`.
2. URL будет `/stih/novyj-stih` (имя файла без `.md`).
3. Заполнить шапку и текст под `---`.

**Шапка (YAML):**

```yaml
---
title: "Название стиха"
chapter: nachalo          # id из content/chapters.ts
order: 10                 # порядок внутри главы
illustration: none        # none | pending | ready
# image: /illustrations/novyj-stih.png   # при ready
featured: false
---
```

**Тело — важно:**

- каждая **строка стиха** — с новой строки в файле;
- **пустая строка** между строфами (иначе строфы сольются).

### 2. Иллюстрация

| `illustration` | Поведение |
|----------------|-----------|
| `none` | Узкая страница только с текстом |
| `pending` | Разворот + тёплая заглушка «художник рисует» |
| `ready` | Разворот + картинка из `public/illustrations/` |

Шаги для готовой картинки:

1. Положить файл в `public/illustrations/`.
2. В шапке: `illustration: ready` и `image: /illustrations/имя.png`.
3. `git push` → Vercel пересоберёт сайт.

### 3. Настройки сайта

`content/site.json`:

```json
{
  "title": "Жизнь напишу карандашом",
  "author": "Андрей Зобнин",
  "subtitle": "собрание стихов",
  "artistName": "Художник",
  "artistNote": "...",
  "stihiUrl": "https://stihi.ru/avtor/boeing75",
  "startSlugs": ["zhizn-napishu-karandashom", "..."]
}
```

| II | `svet` | Свет и любовь |
| III | `ten` | Свет и тень |
| IV | `blizkie` | Близкие |
| V | `doroga` | Дорога |
| VI | `gorod` | Город и день |
| VII | `tajna` | За порогом |
| VIII | `pamyat` | Память |

Пустые главы в оглавлении показывают подсказку «добавьте стихи».

### 5. Нумерация страниц

Сквозной номер листа считается автоматически: `getPoemPageNumber()` в `src/lib/content.ts` — по общему порядку стихов в книге (глава → `order`).

## Дизайн и разворот книги

### Палитра (`src/app/globals.css`)

| Переменная | Назначение |
|------------|------------|
| `--color-parchment` | Фон сайта |
| `--color-parchment-deep` | Разделители |
| `--color-ink` | Основной текст |
| `--color-ink-muted` | Подписи |
| `--color-accent` | Ссылки, колонтитулы |
| `--color-gold` | Орнаменты, номер страницы |

Иллюстрации **не перекрашиваются** — цвет даёт только файл картинки.

### Разворот (десктоп ≥1024px)

Компонент `BookSpread` + стили `.book-outer`, `.book-spread`, `.book-page`, `.book-gutter`.

```
┌ book-edge ─┬──────────────────────────────────────────────┬─ book-edge ─┐
│  (срез)    │  левый лист          │ корешок │ правый лист │   (срез)    │
│            │  иллюстрация         │ закладка│ колонтитул  │             │
│            │  нижний орнамент     │         │ заголовок   │             │
│            │                      │         │ стих        │             │
│            │                      │         │ орнамент + №│             │
└────────────┴──────────────────────┴─────────┴─────────────┴─────────────┘
         drop-shadow под книгой (без тёмной «траурной» рамки)
```

**Принятые решения:**

- обе страницы — «бумага» одного тона;
- переплёт с тёмной лентой и закладкой;
- стих на правом листе: блок по центру страницы, строки выровнены влево внутри колонки;
- **без** строки автора на развороте (как на целевом макете);
- **без** внутреннего скролла на листе — подгонка высоты и `clamp()` для кегля;
- вертикальное центрирование разворота в окне;
- на длинных стихах (например «Черная душа») — более плотная вёрстка на низких экранах.

**Мобильный:** `book-page-mobile` — иллюстрация и текст отдельными блоками.

### Стих без иллюстрации

Класс `.poem-sheet` — заголовок, автор, текст, номер страницы внизу.
